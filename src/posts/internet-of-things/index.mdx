## Internet of Things(IOT)

The Internet of Things, or IoT is the concept of connecting any device (if it has on/off switch) to the internet and the other connected devices to share data and perform specific action. Thanks to the arrival of super cheap computer chips and the ubiquity of wireless networks, it’s possible to turn anything, from something small as a pill to big as aeroplane, kitchen appliances, thermostat, smart microwaves which helps to cook food automatically based on timer, into a part of IoT. Connecting all these different devices and when adding sensors to them adds a level of digital intelligence which enables them to communicate in real-time without involving any human interactions [1] [2].

## Data Generated by IoT

Any physical object can be turned into an IoT device by attaching it to a smart sensor. By 2025, there will be 35 billion [3] IoT devices connecting to the web which is forecasted to create 79.4 [4] Zettabytes of data. It has been roughly calculated by The Atlantic that to hold a Zettabyte we need 1000 data centers which is a lot of data and to process all these we also need to be equipped with modern technology. There is plethora of ways how IoT is generating data. We are surrounded by these devices which has become an integral part of our day to life and many businesses are growing based of IoT devices.

## Brief History of IoT

The concept of devices connected with each other itself dates back in 1832 when the first electromagnetic telegraph was designed. However, the true form of IoT research has begun with the form of world wide web in 1960s. Interestingly, Coca-Cola vending machine situated at Carnegie Melon University and operated by local programmers was the first connected device made. In the late 1990s there has been significant number of inventions done by researchers, but the true significance came by 21st century. ## Technologies behind IoT

Although the idea of IoT devices has been around for so long but it’s the availability of the technologies that made IoT devices an inevitable part of our life.

-   Access to low-cost sensor: Affordable and reliable sensors are making the IoT technology more available to us.
-   Connectivity: A host of network protocols for the internet has made it easy to connect sensors to the cloud and to other “things” for efficient data transfer.
-   Cloud Computing: The access of cloud computing has been an important factor for both businesses and general people to access the cloud and host the data to be shared by different infrastructure and communicate with each other. People can focus more on the business logic rather than maintaining the servers.
-   Machine learning and analytics: Businesses can gather insights faster and more easily by the help of machine learning and analytics along with access to various data sources that are available to us because of IoT devices.
-   Artificial Intelligence: Advances in neural networks have brought NLP to IoT devices (Alexa, Cortana, Siri) and made them affordable and appealing for daily life.

Across the IoT, devices create data that is sent to the main application to consume and used based on the business need. Depending on the device, network and power consumption restraints, data can be sent on real-time, or send in batches of interval or it can be saved locally. However, the real value is restored from the order which data was created. Unless it maintains this feature, it will lose the very aim of the application itself. Whether it is saved in real-time or in time series, traditional way of storing data can be costlier for the businesses. NoSQL framework like Apache Cassandra is better fit for storing IoT data. Due to the volume of write request an IoT devices will make, which Cassandra is capable of even with millions of IoT devices putting write request. Traditional databases have a primary-replica arrangement, where the lead database server will handle all the transactions and synchronously pass them along to other servers if required. This leads to problems in the event of an outage or server failure, as a new primary must be put into place leading to a potential data loss.

For properly configured distributed database systems like Cassandra, there is no primary server that is in charge; each node within a cluster can handle transactions as they come in, and the full record is maintained over time. Even if a server fails, or a node is removed due to loss of network connectivity, the rest of the cluster can continue to process data as it comes in. For time-series data, this is especially valuable as it means that there should be no loss of data in the list of transactions over time.

## Processing IoT Data

Data processing is a must step in data analytics dashboards or applications. It is said that about 80% of the time data scientist use to process the data and rest is used to create machine learning models. Same goes with IoT devices because it generates enormous amount of data, which is necessity to process before undergoing the analytics. However, as the data comes from different sources and devices there are few steps important to apply before starting analytical process.

-   Standardize or transform the data to a unform format ensuring that the data is compatible.
-   Keep backup of the past and newly crated data.
-   Filter out any repetitive data or old data.

There are few architectures [7] that has been followed depending on the use cases to process IoT data because we can not process these data in traditional way. I will discuss them below:

-   **Grid Computing:** Grid computing is a distributed system allowing seamless access to a network of computers connected through a network to produce a specific task. It requires demanding computing infrastructure to process, store and evaluate IoT data. Instead of spending fortune on third party resources, companies can perform grid computing to reduce their budget, but it depends on the team and their goals. In grid computing, data is gathered by central authority who can assign, distribute, and schedule process jobs to different computing machine and later collect the results from sources. Though, it comes with a price, smaller companies won’t be able to perform grid computing unless they have the manpower and budget for this.
-   **Cloud Computing:** Cloud computing has been the most favourable for either storing IoT data or processing it. It costs lower than operating a grid computing system. Most of the cloud providers give automatic scalability option which does not require any configuration on developers end, making it easy to develop the application. IoT can harness powerful cloud-based applications and big data techniques like HBase, MapReduce for processing and analysis [8] specifically for small companies which aren’t ready for upfront investments in Analytics.
-   **Fog Computing:** Cisco has offered a solution architecture named as Fog Computing. The processing is done on the edge devices such as switches, routers, set-top box etc. which are placed at the edge of the local area network. That’s why it also known as edge computing. It’s not implemented to replace cloud but to compliment it. Data is being processed in local area network and then send to the cloud for storage. Depending on the data type and processing different strategies are adopted by Fog Computing [9]. It offers better privacy and security than cloud.
-   **Mobile-edge Computing:** Smart phones are becoming an integral part of IoT in both data source and processing it. Today’s smartphones are equipped with different types of sensors which generates loads of data. Depending on the features it can be processed immediately in mobiles computing power or collect the data and send it to the cloud for future analytics.
-   **On-Site Processing:** On-Site processing is a technique that is used by some IoT devices that are equipped with computing power to process the data it is receiving. This contrast to the other approaches discussed above where data source and computing power does not reside together. Mostly, when data needs to be computed in hard real-time, it uses on-site processing. For example, commercial jets generate 10TB data in an hour which needs to be analysed for auto pilot and other future improvements. Sending this huge amount of data to cloud is not efficient in this use case. Therefore, On-Site processing should be preferable where data does not need to be archived, process centrally and integration with other sources.

## Analyzing IoT data

IoT data is stored mostly to perform different types of analysis on the data it generates. Businesses can grow more, take necessary actions and decision, based on IoT analytics. Several techniques can be used to analyze IoT data.

-   **Prescriptive Analytics:** This technique is used to analyze which steps to take for a specific situation. In commercial applications, prescriptive analytics is used to take decipher large amount of information to perform precise conclusion.
-   **Spatial Analytics:** This method is used to analyze location based IoT data and applications. It analyzes various geographic patterns, spatial relationship, elevation, etc. Parking applications, smart cars, google maps, crop planning all applications of spatial analytics.
-   **Streaming Analytics:** It refers to an event streaming process. Real-time data streams can be analyzed to detect urgent situations to facilitate a real-time response. The types of IoT data that benefit from streaming analytics include those used in traffic analysis, air trafficking, and the tracking of financial transactions.
-   **Time Series Analytics:** It is based on time data to reveal any trends. Our health systems, weather forecasting these are the massive use cases for time series analytics.
